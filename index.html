BeAware Energylife

Rationale and Techniques

http://www.energyawareness.eu/

In a nutshell

  • Sensors in a home measure the electricity usage of various devices.
  • Data is transmitted and stored in a central data store.
  • The Energylife application displays energy usage information in a meaningful way to the occupants of the home.
  • The purpose is to encourage energy saving via information, data visualizations, game mechanics, and community.

Energylife

  • A web application targetted at the iphone.
  • Designed to run full screen.
  • Makes XHR requests to the central data store.
  • All communications use JSON.

Why a web application?

  • Centrally controlled and updateable.
  • Potentially more portable.
  • iPhone support for advanced CSS3 features.
  • Developer preference at the time.

Demo

The Carousel

  • Ability to see all devices at a glance.
  • Easily read visual cues on device cards.
  • Quickly bring interesting ones to the front
  • Each device card is self contained - further information and functionality on the back of the card.

How does it work?

  • iPhone implemented advanced CSS and javascript features not present in other browsers.
    • CSS3 3D transforms
    • CSS3 transitions
    • CSS3 animations
    • Touch events

CSS3 3D transforms

http://www.w3.org/TR/css3-3d-transforms/

  • CSS 3D Transforms extends CSS Transforms to allow elements rendered by CSS to be transformed in three-dimensional space.
  • Should use browser prefixes: -webkit-, -moz-, -o-, -ms-
                transform: <transform function>;
  • transform functions:
    • translateX(x), translate3d(x, y, z)
    • scaleY(y), scale3d(x, y, z)
    • rotateZ(z), rotate3d(x, y, z, angle)
    • perspective(n)
            #transform-demo #box2 {
                -webkit-perspective(1000);
                -webkit-transform: rotateY(45deg) scale3d(1.0,1.5,1.0);
                }

CSS3 transitions

http://www.w3.org/TR/css3-transitions/

  • CSS Transitions allows property changes in CSS values to occur smoothly over a specified duration.
  • Specify a CSS property which should change gradually
  • Specify how long the change should take
                transition-property: <some animatable property>;
                transition-duration: 2s;

CSS3 transitions

  • Some animatable properties:
    • background-color
    • color
    • width, height
    • top, left, bottom, right
    • opacity

CSS3 transitions

            #box3 {
                width: 150px;
                -webkit-transition-property: width;
                -webkit-transition-duration: 3s;
                }
                #box3.transitioned {
                    width: 250px;
                    }
box3

CSS3 transitions in Energylife

  • Transitions are used to flip the card over when it is 'clicked'
  • In this case the animatable property is -webkit-transform
            div#cardRing > div.card > div.cardWrapper {
                -webkit-transition-property: -webkit-transform;
                -webkit-transition-duration: 1.5s;
                }

            div#cardRing > .flipped > div.cardWrapper {
                /* rotate the card 180 degrees along the y-axis. */
                -webkit-transform: rotateY(180deg) scale(2, 2);
                }

CSS3 animations

http://www.w3.org/TR/css3-animations/

  • CSS Animations allow an author to modify CSS property values over time.
  • Declaritively specify named animation effects, without resorting to javascript.

CSS3 animations

            #box4 {
                -webkit-animation-name: rotate-and-move;
                -webkit-animation-duration: 4s;
                -webkit-animation-iteration-count: infinite;
                }

                @-webkit-keyframes rotate-and-move 
                {
                    0%    { -webkit-transform: rotateY(0deg); }
                    50%   { -webkit-transform: rotateY(180deg); }
                    100%  { -webkit-transform: rotateY(360deg); }
                }
#box4

CSS3 animations in Energylife

  • When a device is exceeding it's energy usage target the relevant card pulsates to indicate that there is a problem.
  • Animates opacity
            @-webkit-keyframes pulsate 
            {
                0%    { opacity: 0.5; }
                50%   { opacity: 1; }
                100%  { opacity: 0.5; }
            }

Touch events

http://www.w3.org/TR/touch-events/

  • A set of low-level events that represent one or more points of contact with a touch-sensitive surface.
  • Invented by apple to support multi-touch events on the iphone browser
  • Is in the process of becoming a standard.

Touch events

  • Three main events:
    • touchstart
    • touchmove
    • touchend

Touch events in Energylife

  • Used to rotate the carousel when user "drags".
  • touchstart event
    • track the initial position of the carousel
  • touchmove events
    • track how much the user has moved.
    • rotate the carousel incrementally.
    • also distinguishes it as a "drag" event rather than a "click".
  • touchend
    • rotate the carousel to the nearest card position.

Recap

  • CSS3 3D transforms - change the display of elements in 3D space.
  • CSS3 transitions - make changes to properties into smooth transitions.
  • CSS3 animations - declare animations in CSS rather than code in javascript.
  • Touch events - take advantage of touch screens and multi-touch gestures.
